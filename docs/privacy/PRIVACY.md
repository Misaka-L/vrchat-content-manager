# 隐私政策

*最后更新日期：2026年6月12日*

## 引言

本隐私政策由 VRChat Content Publisher 的开发者（以下简称"我们"）发布，旨在说明当您使用本软件时，我们如何访问、收集、存储、使用和共享（以下统称"处理"）您的数据。

本政策适用于 **VRChat Content Publisher** 桌面客户端（Windows / Linux 版，以下简称"本软件"），包括当您：

- 使用本软件将虚拟形象（Avatars）或世界（Worlds）上传至 VRChat
- 通过本软件登录您的 VRChat 账号

本政策中的"您"指使用本软件的用户。为便于阅读，以下术语在全文中有特定含义：

- **内容**：您在 VRChat 中创建并上传的虚拟形象（Avatars）与世界（Worlds）
- **User Icon（用户头像）**：您的 VRChat 账号个人资料图片，与上传内容无关

## 一、重要提示：非官方客户端与平台风险

本软件是 VRChat 的**第三方、非官方客户端**。我们与 VRChat Inc. 无任何关联，本软件**未获得 VRChat 的授权、认可或背书**。

使用本软件可能违反 VRChat 的服务条款。由此产生的后果——包括但不限于您的 VRChat 账号被警告、限制功能、暂停或永久封禁——**均由您自行承担**。开发者对此类平台处罚及由此产生的任何损失不承担任何责任。

## 二、我们收集哪些数据？

### 2.1 您主动提供给我们的数据

当您登录时，您需要提供：

- VRChat 用户名
- VRChat 密码
- TOTP 动态验证码（如您已启用双重验证）

**密码和验证码仅用于完成当次登录验证，使用后立即丢弃，不会以任何形式存储在本软件或您的设备上。**

### 2.2 我们从 VRChat 平台获取的数据

登录成功后，本软件直接调用 VRChat 的服务器接口获取以下信息。这些接口的文档由社区通过逆向工程整理（VRChat 未提供官方 API 文档），但所有数据请求均由您的设备直接发往 VRChat 服务器：

**用户基本信息**（登录时获取，存储在 `sessions.json`）：

- 用户 ID
- 用户名
- User Icon（用户头像）
- 昵称

**内容信息**（发起上传任务时获取，存储在 `database/app.db`）：

- 您准备上传的内容的名称与 ID

这些信息仅用于实现本软件的核心功能，登出或删除任务时相应数据将被清除。详见第 5.2 节。

VRChat 对其平台数据的收集和处理受其自身隐私政策约束，详情请参阅 [VRChat 隐私政策](https://hello.vrchat.com/privacy)。

### 2.3 您指定上传的内容文件

当您发起上传任务时，本软件会读取您指定的 AssetBundle 文件（由 VRChat SDK 构建），并在操作系统缓存目录中暂存副本，用于上传至 VRChat 服务器。上传任务完成或您主动删除任务后，相关缓存将被清除。

### 2.4 我们自动生成的数据

为维持软件正常运行，本软件会在您的设备上自动生成以下数据：

- **登录 Cookies**：用于保持您与 VRChat 的登录会话，存储在 `sessions.json`
- **应用设置**：您偏好的配置选项（包括 Sentry 遥测偏好），存储在 `settings.json`
- **任务记录**：您的上传任务状态与进度，存储在 `database/app.db`
- **本地调试日志**：用于排查软件问题（详见第 2.5 节），存储在 `logs/`

### 2.5 遥测数据与本地调试日志

本软件会生成两类记录，二者的处理方式截然不同：

**Sentry 遥测数据（可控制）**

当您启用 Sentry 遥测时，崩溃报告和诊断数据会根据您选择的模式（详见第 4.1 节）直接发送至 Sentry 运营方 Functional Software, Inc.。在默认的"隐私模式"下，用户 ID 等标识符在离开您的设备之前会经过脱敏处理（详见第 4.1 节），原始值在哈希后立即丢弃，不会上传。

**本地调试日志（完全由您掌控）**

存储在您设备上的本地日志文件可能包含**未脱敏的原始数据**（如完整的用户 ID、内容名称等），这是为了在排查复杂问题时能够准确定位原因。

**本地调试日志永远不会被自动上传。** 开发者仅在您主动通过 GitHub Issues 或邮件发送这些文件时才能查看。

### 2.6 数据收集概览表

| 数据类别 | 是否储存 | 存储位置 | 是否上传 | 您可否控制 |
|---------|---------|---------|---------|-----------|
| 密码 / TOTP | 否 | — | — | — |
| Cookies | 是，明文 | `sessions.json`（本地） | 否 | 登出即清除 |
| 用户基本信息（用户 ID、用户名、User Icon、昵称） | 是，明文 | `sessions.json`（本地） | 否 | 登出即清除 |
| 内容信息（名称、ID） | 是 | `database/app.db`（本地） | 否 | 删除任务即清除 |
| AssetBundle 文件 | 是（上传期间暂存） | 操作系统缓存目录 | 是（仅上传至 VRChat 服务器） | 删除任务即清除缓存 |
| Sentry 遥测数据 | 是 | 存储在 Sentry 服务器（德国） | 视模式而定 | 可随时禁用或切换模式 |
| 本地调试日志 | 是，明文 | `logs/`（本地） | **永不自动上传** | 完全自主管理 |

## 三、我们如何使用这些数据？

### 3.1 提供核心功能

本软件的核心功能是为您提供 VRChat 内容上传服务。为实现这一服务，您的数据会被用于：

- **登录验证**：密码和 TOTP 验证码仅用于当次登录，使用后立即丢弃。
- **维持会话**：Cookies 与用户基本信息保存在本地，避免每次启动都需要重新登录。
- **执行上传**：将您指定的 AssetBundle 文件上传至 VRChat 平台。
- **恢复上传进度**：上传任务的相关信息会保存在本地。如果上传过程中软件意外退出，重启后可以从中断处继续。

### 3.2 崩溃诊断与改进

- **Sentry 遥测**：在您启用 Sentry 的情况下，崩溃报告和诊断数据用于帮助开发者定位软件缺陷、分析性能问题。Sentry 服务器位于德国（欧盟）。
- **本地调试日志**：仅存储在您的设备上。当您报告问题时，开发者可能会请求您手动分享相关日志，以便深入排查。**是否分享完全由您决定。**

## 四、我们如何与第三方共享数据？

我们不会出售您的数据。仅在以下情形中，部分数据会被直接传输至第三方服务。每次传输均受该第三方自身隐私政策的约束，我们不对第三方的数据处理行为负责。

### 4.1 崩溃诊断服务商 Sentry

当您启用 Sentry 遥测时，崩溃诊断数据会被直接发送至 **Functional Software, Inc.（Sentry 服务的运营方）**，用于帮助我们定位和修复软件问题。数据存储在位于德国的服务器上。Sentry 作为独立的数据处理方，对这些数据的使用受其 [隐私政策](https://sentry.io/privacy/) 约束。

**三种模式**

您在首次启动本软件时会被询问 Sentry 偏好，此后可随时在设置中更改。

> 默认启用隐私模式是因为它能在保护您隐私的同时提供足够的诊断信息。完整模式仅在开发者需要更详细的数据以排查特定复杂问题时，由您自主选择是否临时开启。

| 模式 | 发送的数据 |
|-----|-----------|
| 禁用 | 无 |
| 隐私模式（默认） | 设备信息、崩溃堆栈、经脱敏处理的标识符 |
| 完整模式 | 隐私模式的所有数据 + 原始标识符（未脱敏）+ IP 地址 |

**脱敏处理细节**

在隐私模式下，以下标识符在离开您的设备前会经过脱敏处理：

- **脱敏对象**：用户 ID、世界 ID、虚拟形象 ID
- **脱敏方法**：MD5 哈希 + 加盐 + 截取前八位
- **示例**：`usr_c6806bef-b885-427d-acaf-e24a66acb829` → `usr_2af9d47c`
- **用户名及内容名称**：直接替换为占位符，不上传原文
- **原始数据处理**：原始 ID 在哈希计算完成后立即丢弃，不离开您的设备

**重要提醒**：无论选择何种模式，一旦数据上传至 Sentry，将无法从 Sentry 服务器上删除。

### 4.2 VRChat 平台

本软件作为客户端运行，您在使用本软件时产生的网络请求由您的设备直接向 VRChat 服务器发起。VRChat 对从您设备接收的数据的处理，受其 [隐私政策](https://hello.vrchat.com/privacy) 约束。我们与 VRChat 之间不存在独立的数据共享关系。

### 4.3 其他数据

除上述情形外，我们不会向任何其他第三方出售、出租、交易或共享您的数据。

## 五、数据如何存储？安全吗？

### 5.1 存储位置

所有本地数据存储在您的设备上：

- **Windows**：`%LOCALAPPDATA%\vrchat-content-manager-81b7bca3\`
- **Linux**：`$XDG_CONFIG_HOME/vrchat-content-manager-81b7bca3/`（大部分发行版会储存在 `$HOME/.config` 下）

### 5.2 存储格式

所有文件以**明文（未加密）**格式存储：

- `sessions.json`：登录 Cookies、用户基本信息（用户名、用户 ID、User Icon、昵称等）
- `settings.json`：应用设置（含 Sentry 偏好）
- `database/app.db`：任务记录（上传任务信息）
- `logs/`：调试日志文件（纯文本）
- 操作系统缓存目录：上传过程中暂存的 AssetBundle 文件，任务完成或删除后自动清除

### ⚠️ 重要安全提醒

**Cookies 以明文形式存储在您的设备上。任何能够访问您设备文件系统的人或恶意软件都可能读取这些 Cookies 并用于登录您的 VRChat 账号。** 我们建议您：

- 不要与他人共享您的计算机用户账户
- 在公用或共享设备上使用本软件后及时登出
- 保持操作系统的安全更新

### 5.3 卸载后的数据残留

卸载本软件不会自动删除上述数据文件夹。如需彻底清除所有本地数据，请手动删除对应路径下的 `vrchat-content-manager-81b7bca3` 目录。我们计划在未来版本中提供卸载时的清理选项。

## 六、您拥有哪些选择与权利？

- **登出**：点击"登出"后，`sessions.json` 中的 Cookies 及用户基本信息将被立即清除。
- **彻底清除数据**：手动删除配置目录（路径见第 5.1 节）即可移除所有本地存储的数据。
- **控制 Sentry 遥测**：您可以在设置中随时在"禁用""隐私模式""完整模式"之间切换。选择"禁用"将停止所有遥测数据上传。
- **管理本地调试日志**：日志完全由您掌控。您可以随时查看、删除或选择不分享这些文件。本软件不会自动上传它们。

**需要注意的限制**：

- 一旦遥测数据被上传至 Sentry，我们无法协助您从 Sentry 服务器上删除这些数据。您可以通过禁用遥测来阻止未来的数据上传，但这不影响已经发送的数据。
- VRChat 平台未提供"撤销其他登录会话"的功能，因此您无法通过本软件强制登出其他设备上的 VRChat 会话。
